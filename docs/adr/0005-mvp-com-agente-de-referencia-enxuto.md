# MVP com agente de referência enxuto

O MVP deve validar a estrutura do runtime gerado com um agente de referência simples, parecido com uma entrevista guiada, mas sem reproduzir toda a complexidade funcional da API ProUp. Esse agente deve exercitar banco PostgreSQL, configuração de OpenAI, chamada real ou mockável de LLM, prompts em Markdown, safety simplificado, retries, erros, fluxo de dados, sessões e turnos, para provar a arquitetura antes de avançar para agentes ricos como AAP ou COPSOQ-Pulso.
